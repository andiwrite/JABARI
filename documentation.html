How I Built This Website (Jabari)

Why the Name "Jabari"?

I chose the name "Jabari" because it means "brave" or "fearless" in Swahili. I wanted the website to stand out and represent strength, reliability, and a bold approach to technology and e-commerce. The name is unique and easy to remember, which is great for branding.

Languages Used

  • HTML5 – for the structure of all the web pages.
  • CSS3 – for styling, layout, and making the site look modern and clean.
  • JavaScript – for interactive features like the cart, timers, and animations.

I used only these core web languages so the site loads fast and works everywhere.

Plugins and Libraries

  • Bootstrap 5: This is a CSS framework that helps with layout, grids, and responsive design. It made it much easier to make the site look good on phones, tablets, and computers without writing a lot of custom CSS.
  • Font Awesome: This is a library for icons (like the cart, user, and social media icons). It makes the site look more professional and user-friendly.

What is Bootstrap?

Bootstrap is a popular open-source CSS framework that provides ready-made styles, layout grids, and components for building responsive and modern websites. It helps developers quickly create layouts that look good on all devices (phones, tablets, computers) without having to write a lot of custom CSS from scratch.

How Bootstrap Was Used in This Site

  • The responsive grid system (rows and columns) to arrange products, categories, and sections.
  • Navigation bars, buttons, forms, and pagination for a consistent look and feel.
  • Making the site automatically adjust to different screen sizes.
  • Utility classes for spacing, alignment, and colors, which made the design process faster and more reliable.

What Was Done With the JavaScript Code

  • The main JavaScript file (cart.js) was kept as originally written, handling cart management, cart icon updates, and interactive cart features.
  • Inline JavaScript in the HTML files manages animations, countdown timers, wishlist toggling, sorting, and view switching.
  • No JavaScript was removed or broken during recent edits; all interactive features remain functional.
  • No new JavaScript was added as part of the recent changes.

How I made the site responsive and scalable

I wanted everyone to have a good experience, whether they're on a phone, tablet, or computer. So, I used Bootstrap's grid system, which lets me arrange things in rows and columns that automatically adjust to the screen size. I also added media queries in the CSS, so things like the menu, cards, and images change their size or layout on smaller screens.

For scalability, I kept the code organized and used reusable components like the header and footer on every page. I used flexible units (like %, rem, and em) so the design grows or shrinks smoothly if you zoom in or out. This way, the site can easily handle more products or new features in the future without breaking the layout.

JavaScript Features and Algorithms

  • Cart Functionality: I wrote JavaScript to let users add items to the cart, change quantities, and remove items. The cart totals update automatically when you change the quantity or delete a product. This is done by listening for button clicks and updating the HTML and totals in real time.
  • Countdown Timers: The timers for deals and flash sales use JavaScript intervals to count down and update the time left every second.
  • Animations: I used Intersection Observer in JavaScript to animate cards and sections as you scroll down the page. This makes the site feel more dynamic and modern.

All the JavaScript is written in a simple way so it's easy to understand and change later.

My Process

  1. I started by planning the main pages and features I wanted (like Home, Products, Cart, Support, etc.).
  2. I designed the layout using Bootstrap and made sure the navigation and footer were consistent everywhere.
  3. I added real product images and made sure all links go to the right pages.
  4. I wrote the JavaScript for the cart and other interactive features.
  5. I tested the site on different devices to make sure it's responsive and easy to use.

Summary

This website was built to be fast, modern, and easy for anyone to use. I used only the most important tools and kept the code simple so it's easy to update or expand in the future. If you want to learn more or see the code, feel free to reach out!
